# Training Planner — План тренировок

**Автор:** Петрова Диана
**Дата создания:** май 2026

## Описание
Приложение с графическим интерфейсом (tkinter) для планирования и учёта тренировок. 
Позволяет вести журнал тренировок, фильтровать записи, сохранять данные в JSON и загружать их при запуске.

---

## Возможности кода

### 1. Управление тренировками
- **Добавление тренировки** 
  Поля: дата, тип тренировки (выпадающий список), длительность в минутах.
- **Удаление тренировки** 
  Удаление выбранной строки из таблицы с подтверждением.
- **Автоматическая нумерация** 
  Каждой тренировке присваивается уникальный ID.

### 2. Валидация ввода
- **Дата** — проверка формата `ГГГГ-ММ-ДД` (например, `2025-05-06`). 
  Неверный формат → сообщение об ошибке.
- **Длительность** — проверка, что введено положительное число (целое или дробное). 
  Отрицательные значения, ноль или текст → ошибка.

### 3. Отображение данных
- **Таблица (Treeview)** с колонками: 
  `ID`, `Дата`, `Тип`, `Длительность (мин)`.
- **Сортировка** — тренировки автоматически сортируются по дате.
- **Прокрутка** — вертикальный скроллбар для таблицы.

### 4. Фильтрация
- **По типу тренировки** 
  Выбор из списка: Все, Бег, Плавание, Велоспорт, Силовая, Йога.
- **По точной дате** 
  Ввод даты в формате `ГГГГ-ММ-ДД`. 
  При неверном формате — предупреждение, фильтр не применяется.
- **Сброс фильтров** — кнопка для очистки обоих фильтров.

### 5. Работа с JSON
- **Автосохранение** — после добавления или удаления тренировки данные сразу сохраняются в `workouts.json`.
- **Ручное сохранение** — кнопка «Сохранить в JSON».
- **Ручная загрузка** — кнопка «Загрузить из JSON».
- **Загрузка при старте** — если файл `workouts.json` существует, данные автоматически загружаются.
- **Кодировка UTF-8** — корректная работа с русскими названиями.

### 6. Интерфейс
- **Поля ввода** с подсказками (placeholder для даты).
- **Выпадающий список** для типа тренировки (нельзя ввести произвольное значение).
- **Кнопки с иконками** (эмодзи): 
  ➕ Добавить, 🔍 Применить фильтр, ❌ Сбросить, 💾 Сохранить, 📂 Загрузить, 🗑 Удалить.
- **Группировка элементов** — рамки LabelFrame для логического разделения.

### 7. Обработка ошибок
- Сообщения `messagebox` при:
  - Неверной дате
  - Неверной длительности
  - Попытке удаления без выбора
  - Ошибках чтения/записи JSON
- Подтверждение удаления (yes/no).

### 8. Дополнительные возможности
- **Автоподстановка текущей даты** при запуске.
- **Очистка поля длительности** после добавления.
- **Сохранение данных в человекочитаемом JSON** (с отступами `indent=4`).
